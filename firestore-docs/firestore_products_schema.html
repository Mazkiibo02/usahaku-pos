Autocomplete search — tanpa biaya tinggi
1 Tokenisasi saat write 0 read cost
Saat produk disimpan, Cloud Function (atau client) generate array prefix:
"Kaos Polos" → ["k","ka","kao","kaos","kaos ","kaos p","kaos po","kaos pol","kaos polo","kaos polos"]
+ tokenisasi setiap kata: ["p","po","pol","polo","polos"]. Simpan ke field nameTokens.
2 Query dengan array-contains 1 read per doc match
db.collection('products')
 .where('tenantId','==', tid)
 .where('isActive','==', true)
 .where('nameTokens','array-contains', input.toLowerCase())
 .limit(10)

Hanya dokumen yang match yang dibaca. Input user 3 karakter sudah cukup untuk memulai query.
3 Debounce + local cache 0 extra reads
Di frontend: debounce(300ms) sebelum fire query. Cache hasil terakhir di memory. Jika user ketik "kao" lalu "kaos", hasil "kaos" sudah di cache — tidak perlu re-query jika kata sebelumnya sudah match.
! Barcode scan (instant) 1 read
Query by barcode tidak perlu nameTokens. Simpan flat index terpisah barcodeIndex/{barcode} → {productId, skuId, tenantId}. Barcode scan = 1 read di collection kecil + 1 read dokumen produk = 2 reads total.
Composite indexes yang dibutuhkan
Catalog query required
tenantId ASC, isActive ASC, updatedAt DESC
Untuk: load semua produk aktif milik tenant, sorted terbaru.
Autocomplete query required
tenantId ASC, isActive ASC, nameTokens ARRAY
Untuk: search produk aktif per tenant dengan prefix token.
Category filter optional
tenantId ASC, category ASC, isActive ASC
Untuk: tampilkan produk per kategori (fashion, fnb, dsb).
Security rules skeleton
// firestore.rules
match /products/{productId} {
  allow read: if isAuthenticated()
    && resource.data.tenantId == getUserTenantId();

  allow write: if isAuthenticated()
    && request.resource.data.tenantId == getUserTenantId()
    && hasRole(['admin', 'manager']);
}
